## Задание: чтение файла по частям с сохранением резюме

### 1. Инициализация
- Определи путь к файлу: `temp/671.md`
- Определи количество строк в файле (используй `size_chunk`).
- Установи переменную `start_line = 0`.
- Если в промежуточном хранилище уже есть ключ `progress`, прочитай из него `start_line` и продолжай с него.

### 2. Цикл обработки
Повторяй шаги 2.1–2.4, пока не будет прочитан весь файл.

#### 2.1. Прочитай очередной чанк
- Используй `view_chunk` с параметрами:
  - `path`: путь к файлу
  - `start_line`: текущее значение
  - `lines`: 100
  - `max_chars`: 10000
- Из результата возьми `startLine`, `endLine`, `chunk`.

#### 2.2. Составь краткое резюме
- На основе прочитанного `chunk` составь краткое резюме (1–3 предложения), описывающее ключевое содержание.
- Сохрани резюме в промежуточное хранилище с ключом `summary_part_N`, где N — номер части (начинай с 1 и увеличивай после каждого чанка). Используй `intermediate_pad` с параметрами:
  - `label`: `summary_part_N`
  - `data`: текст резюме
  - `description`: краткое пояснение (опционально)

#### 2.3. Обнови состояние
- Вычисли следующий `start_line = endLine + 1`.
- Сохрани новое значение `start_line` в промежуточное хранилище по ключу `progress` (можно использовать `intermediate_pad` с тем же ключом, затирая предыдущее значение).

#### 2.4. Проверка завершения
- Если `start_line >= totalLines` (общее количество строк получено на шаге 1), то файл прочитан полностью. Переходи к пункту 3.
- В противном случае продолжай цикл (возвращайся к 2.1).

### 3. Завершение
- Выведи итоговый отчёт: сколько частей обработано, общее количество строк.
- Удали временный ключ `progress` (опционально).

### 4. Формат ответов на ключевые вопросы
На любые вопросы пользователя, которые соответствуют пунктам 5 и 6 исходного списка, отвечай строго по формату:

- **«Выведи свой статус. Какое у тебя задание?»**  
  Ответ должен быть кратким (не более 3 предложений), перечисляющим текущий прогресс: сколько строк обработано из скольких всего, номер текущего чанка, следующий `start_line`.

- **«Файл прочитан полностью?»**  
  Отвечай только **ДА** или **НЕТ**.

- **«Если НЕТ, то перейди к пункту списка по номеру 4»** — этот пункт не требуется обрабатывать вручную, так как цикл автоматический. Просто продолжай выполнение цикла.

### 5. Примечания
- Не создавай промежуточные ключи для резюме заранее — только по мере обработки.
- Не используй `todo_goto` — управляй состоянием через промежуточное хранилище.
- Все вызовы инструментов (`size_chunk`, `view_chunk`, `intermediate_pad`) должны выполняться по необходимости, без лишних дублирований.
- Если в процессе произошла ошибка чтения, остановись и сообщи об ошибке.
