3. На основе результата аудита вызови skill `refactor-plan` и сформируй реалистичный план рефакторинга с оценкой сложности и рисков для ветки $branch.
4. Сформулируй краткий summary для отчёта (3–5 предложений), который можно отправить в командный чат или добавить в заметки пользователя $user за дату $date.
