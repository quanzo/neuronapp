Тебе передан файл по пути `$path`.
Если указан параметр `$focus`, уделяй ему особое внимание.

Проанализируй содержимое файла и подготовь аудит по разделам:
1. **Назначение файла** — кратко опиши, за что он отвечает.
2. **Стиль и читаемость** — именование, структура, избыточность, соблюдение принятых соглашений.
3. **Архитектура** — насколько логично разнесены обязанности, нет ли «божественных» классов/функций.
4. **Потенциальные баги и edge cases** — граничные условия, обработка ошибок, null/пустых значений, конкурирующие изменения.
5. **Производительность** — возможные узкие места, тяжёлые циклы, неэффективные операции ввода-вывода.
6. **Безопасность** (если применимо) — работа с вводом пользователя, файлами, shell-командами, сетевыми запросами.
7. **Рекомендации** — 5–10 конкретных шагов по улучшению кода с приоритезацией (high/medium/low).

Если по каким‑то пунктам информации явно не хватает, так и напиши, не додумывая детали.
