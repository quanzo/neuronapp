На входе у тебя есть результат аудита кода (выполненный отдельным skill) и, при необходимости, общий обзор проекта.

Задача:
1. Сгруппировать найденные проблемы по типам (архитектура, стиль, производительность, безопасность, тестирование).
2. Составить реалистичный план рефакторинга из 5–10 шагов.
3. Для каждого шага указать:
   - цель;
   - примерные затраты (S/M/L, где S — до пары часов, L — несколько дней);
   - риск (низкий/средний/высокий) и возможные побочные эффекты;
   - зависимости от других шагов (если есть).
4. В конце выделить 2–3 шага, которые дадут максимальную отдачу при минимальных рисках.

Формат ответа:
- Короткое резюме (2–3 предложения).
- Список шагов рефакторинга в виде нумерованного списка.
- Блок "Приоритет на ближайший спринт" с 2–3 ключевыми задачами.
