Ты получишь текстовый план задания в свободной форме.
Твоя задача — превратить его в нумерованный список задач (todolist) для пошагового выполнения.

Правила строгого формата:
- Ответ должен содержать ТОЛЬКО нумерованный список, без вступительных фраз, комментариев, обрамления в тройные кавычки или другие блоки.
- Каждый пункт начинается с новой строки с номера, точки и пробела, например: «1. », «2. », «3. ».
- Первая строка пункта — это краткое название задачи.  
  Следующие строки (могут быть несколько) — максимально подробное описание: что именно нужно сделать, какой ожидается результат.
- Первый пункт обязательно должен включать краткое изложение всей исходной задачи и данные, необходимые для начала (инициализация).
- По возможности добавляй пункты для проверки промежуточных результатов и исправления ошибок.
- Никакой другой информации в ответе быть не должно. Список — это единственное, что ты выводишь.

Пример структуры (текст внутри — только иллюстрация):

1. Общая задача и исходные данные
Здесь кратко описана задача и всё, что дано для старта.
2. Подготовка окружения
Установить необходимые библиотеки, проверить версии, настроить пути.
Результат: окружение готово, импорты работают.
3. Загрузка и проверка данных
Загрузить датасет из указанного источника, проверить размер, типы колонок, наличие пропусков.
Результат: данные загружены, пропуски выявлены.
4. Проверка качества данных и исправление ошибок
Проверить корректность данных согласно требованиям, обработать пропуски, удалить дубликаты.
Результат: чистые данные, готовые к анализу.

Обрати внимание:
- Номер, точка, пробел — строго.
- Описание задачи может занимать несколько строк, но все они относятся к текущему номеру, пока не встретится следующий номер.
- Никакого оформления markdown-блоками (кроме самого списка) не используй.

Теперь преобразуй этот план в todolist:

$unformattedplan
