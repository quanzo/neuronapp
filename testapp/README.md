# testapp2 — тестовое окружение для LLM‑агентов

Директория `testapp2` используется как отдельный `APP_WORK_DIR` для экспериментов с LLM‑агентами
и не требует изменения основного кода приложения.

## Структура

- `config.jsonc` — базовая конфигурация приложения (userId, настройки context_files).
- `agents/`
  - `agent-dev-coder.php` — агент‑программист, умеет смотреть/искать/править файлы и выполнять безопасные shell‑команды.
  - `agent-researcher.php` — агент‑исследователь, работающий с wiki/search инструментами.
  - `agent-maintenance.php` — агент для ежедневных брифов и базовой диагностики проекта.
- `skills/` — переиспользуемые текстовые навыки:
  - `project-overview.md` — краткий обзор проекта и рисков.
  - `file-audit.md` — аудит качества кода файла.
  - `refactor-plan.md` — план рефакторинга по результатам аудита.
  - `daily-brief-project.md` — ежедневный бриф по активности в проекте.
- `todos/` — сценарии полезной работы:
  - `file-audit-run.md` — полный цикл "аудит файла → план рефакторинга → summary".
  - `daily-brief-project.md` — ежедневный бриф по проекту.
  - `topic-research-wiki.md` — исследование темы через wiki‑поиск.
- `docs/` — файлы контекста, доступные через `@docs/...`:
  - `project-notes.md` — заметки о целях тестового окружения.
  - `code-style.md` — краткие правила код‑стайла и тестирования.
- `.sessions/`, `.logs/`, `.store/` — служебные директории (история диалогов, логи, чекпоинты todolist).

## Предварительные условия

1. Переменная окружения `APP_WORK_DIR` должна указывать на директорию `testapp2`.
2. В `config.jsonc` включено автоподключение файлов контекста:
   - `context_files.enabled = true`;
   - `context_files.max_total_size = 1048576`.

## Запуск сценариев todolist

Все команды выполняются из корня проекта (где находится `bin/console`).

### 1. Аудит файла и план рефакторинга

- Агент: `agent-dev-coder`
- Сценарий: `file-audit-run`

```bash
php bin/console todolist --todolist file-audit-run --agent agent-dev-coder
```

Ход работы:
- агент попросит указать путь к файлу и контекст (что беспокоит);
- выполнит аудит файла через skill `file-audit`;
- на основе аудита построит план рефакторинга через skill `refactor-plan`;
- сформирует краткое summary для отчёта.

### 2. Ежедневный бриф по проекту

- Агент: `agent-maintenance`
- Сценарий: `daily-brief-project`

```bash
php bin/console todolist --todolist daily-brief-project --agent agent-maintenance
```

Ход работы:
- агент уточнит, что делалось за день;
- использует заметки `@docs/project-notes.md` и, при наличии, логи из `.logs`;
- с помощью skill `daily-brief-project` сформирует разделы "Что сделано", "Проблемы и риски", "Следующие шаги";
- выдаст краткое резюме для командного чата.

### 3. Исследование темы через wiki

- Агент: `agent-researcher`
- Сценарий: `topic-research-wiki`

```bash
php bin/console todolist --todolist topic-research-wiki --agent agent-researcher
```

Ход работы:
- агент уточнит тему и цель исследования;
- сформирует несколько запросов и выполнит их через инструменты wiki‑поиска;
- подготовит структурированный обзор (определение, ключевые факты, подходы, подводные камни);
- предложит, как применить знания к текущему проекту.

## Использование файлов контекста

Во входных текстах skills и todolist можно указывать @‑ссылки на файлы, например:

- `@docs/project-notes.md`
- `@docs/code-style.md`

При включённом `context_files.enabled` эти файлы будут автоматически прикреплены как вложения
и попадут в контекст LLM‑агента при выполнении навыка или todo‑шага.
