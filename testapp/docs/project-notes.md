# Project notes (testapp)

Эта директория выступает как контекст для тестового приложения `testapp`, используемого LLM‑агентами.

## Цели тестового окружения

- Проверить работу инфраструктуры агентов (`agent-dev-coder`, `agent-researcher`, `agent-maintenance`) без изменения основного приложения.
- Отработать сценарии:
  - аудит и рефакторинг отдельных файлов;
  - исследование тем через wiki/поиск;
  - ежедневные брифы по активности в проекте.
- Продемонстрировать использование skills и todolist‑сценариев вместе с инструментами (`ViewTool`, `GrepTool`, `GlobTool`, `EditTool`, `BashTool`, wiki‑search и др.).

## Ключевые артефакты в `testapp`

- `agents/` — конфигурации агентов LLM для разных задач.
- `skills/` — текстовые навыки, которые превращаются в инструменты и переиспользуются в сценариях.
- `todos/` — сценарии todolist, описывающие полезную работу (аудит, обзор, брифы).
- `.sessions/`, `.logs/`, `.store/` — служебные директории для истории чатов, логов и чекпоинтов.

## Типовые задачи для агентов

- Подготовить обзор по модулю или файлу кода.
- Сформировать план рефакторинга по результатам аудита.
- Собрать информацию по теме с использованием wiki и оформить короткий отчёт.
- Сформировать ежедневный бриф по активности в проекте и предложить следующие шаги.
