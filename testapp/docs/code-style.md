# Code style notes (testapp)

Эти заметки предназначены для LLM‑агентов и людей, работающих в тестовом окружении `testapp`.
Они не заменяют основную документацию проекта, а служат кратким напоминанием.

## Общие принципы

- Код должен быть **читаемым и самодокументируемым**.
- Повторяющаяся логика выносится в отдельные хелперы или сервисы.
- Для структурированных данных используем **DTO‑классы**, а не «сырые» массивы.

## PHP

- Стиль: PSR‑12 (в проекте используется `phpcs` с соответствующим конфигом).
- Каждый класс должен иметь краткое описание назначения.
- Методы и функции документируются через PHPDoc:
  - назначение;
  - параметры и типы;
  - возможные исключения и особые случаи.
- Сеттеры — во fluent‑стиле (возвращают `$this`).

## Тесты

- Обязательно проверять граничные условия.
- В тестовых наборах должны быть заведомо некорректные данные, чтобы проверить реакцию проверяемого кода.
- Минимальный набор тестовых данных для нетривиальных случаев — около 10 вариантов.
- Каждый тест снабжается кратким комментарием о том, что именно он проверяет (в тестовом коде).

## Bash и инструменты

- Любые действия через shell‑инструменты должны быть **безопасными по умолчанию**:
  - не использовать разрушительные команды (`rm -rf`, прямые перезаписи без бэкапа и т.п.);
  - отдавать предпочтение командам чтения/анализа (`git status`, `git diff`, `composer show`).
- Для автоматических правок файлов использовать цепочку:
  1. `GlobTool` / `GrepTool` — поиск мест правок;
  2. `ViewTool` — просмотр с нумерацией строк;
  3. `EditTool` — точечная замена с бэкапом.
