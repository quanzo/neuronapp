# Создаем TodoList - список заданий на исполнение

Язык программирования: php 8.1

# Структура каталогов

Все пути к каталогам заданы от корня проекта

* `classes/` - здесь будем хранить классы
* `interfaces/` - здесь будут интерфейсы

Базовый namespace для классов `app\modules\neuron\classes`

Базовый namespace для интерфейсов `app\modules\neuron\interfaces`

Все классы для реализации текущего проекта разместить в `classes/todo` 

# Структура

Каждое задание в списке является экземпляром классом `Todo`

Список заданий реализуется в классе `TodoList`

## класс Todo

Каждое задание - это текст. Статичный конструктор принимает текст и возвращает экземпляр класса.

Реализует интерфейс `ITodo` с методом `getTodo()`

## класс TodoList

Список экземпляров класса `Todo`. Реализует интерфейс `ITodoList`.

### Методы:

`pushTodo` - помещает Todo в список. На входе один или несколько объектов класса Todo

`popTodo` - извлекает один элемент из списка по методу FIFO

`getOptions` - возвращает массив с опциями

### Конструктор класса

Конструктор принимает текст и на его основании формриует список заданий на исполнение.

#### Формат входного текста:

```text
-----
option1: option1Value
option2: option2Value
-----
1. Первое задание

Задание продолжается

2. Второе задание

Продолжение второго задания

```

Допускается отсутствие блока опций и/или блока с заданиями.

##### Опции

Первым блоком идут опции списка заданий. Они отделены строкой `-----`. Символов `-` может быть больше.

Каждая опция начинается с имени, затем двоеточие и значение опции. Значение опции может соответствовать формату json.

##### Задание

__Допускаются пустые строки перед текстом первого задания.__

Каждое задание начинается с новой строки и числа, эатем точка и пробел. Задание может быть многострочным.

Задание может быть одно в списке и тогда может не иметь номера.
