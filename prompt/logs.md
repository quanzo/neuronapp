# логирование

Логирование по стандарту __PSR-3__

`APP_WORK_DIR` будет папка `.logs` - вот в ней будут файлы логов. В `ConfigurationApp` определить статичный метод, который вернет короткое имя и полный путь к каталогу логов.

Имя файла лога будет соответствовать имени сессии. Для агентов - имя сессии агента. В `ConfigurationApp` добавить свойство `debug` которое будет включать вывод определнного уровня отладочных сообщений или отключать вывод логов.

Добавить в проект логирование ошибок, когда какой то элемент не найден или задан неправильно; логировать вызовы инструментов и/или skill; логировать исполенение todolist. В диагностических сообщениях дописывать время исполнения.

Экземпляр логгера должен создаваться в ConfigurationApp и передаваться всем заинтересованным классам через конструктор или сеттер. Если объект логгера не передан, то просто не будет записи в лог - ошибок быть не должно.


# Консольная команда для отправки сообщения агенту и получения ответа

Команды храним в папке `src/classes/command`

Имя команды: simplemessage

Команда должна передать LLM, на которую сконфигурирован агент, сообщение и вывести результат. Результат - это ответ агента и `sessionKey` чтобы сессию можно было продолжить.

Пример: `--agent default --message "Hello default agent! How are you!"`

Параметры:
- agent - агент LLM который будет использован для обработки
- message - сооббщение
- session_id - опционально можно передать `sessionKey` в виде строки в том формате, в котором метод `buidSessionKey` и тогда сессия должна быть продолжена. Если опция не передана, то создаем новую. Если опция передана, но ее нет - то выдать сообщение об ошибке.

Примечание: для исполнения можно использовать `TodoList` с одним пунктом.
