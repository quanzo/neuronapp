# Правила

* Документируй код: все классы должны иметь заголовок с описанием предназначения класса; все методы и функции должны иметь заголовок с описанием предназначения, описание функционала, и документированные в phpdoc параметры.
* Не используем структуры в виде массивов - для структурированной информации создаем DTO.
* Если есть повторяющийся функционал и он не привязан к объекту, то вынести его в хелпер.
* В конце сессии, после внесения изменений запусти линтер, phpstan и тесты.

## Каталоги

* `src/classes/command` - реализация классов консольных команд
* `src/classes/config` - классы для работы с настройками и конфигурациями
* `src/classes/dto` - DTO
* `src/classes/neuron` - классы для переопределения поведения элементов NeuronAI
* `src/classes/producers` - производители или фабрики элементов
* `src/classes/skill` - навыки
* `src/classes/todo` - списки заданий
* `src/classes` - корневой каталог классов
* `src/enums` - перечисления enums
* `src/helpers` - хелперы здесь храним
* `src/interfaces` - определения интерфейсов
* `src/traits` - трейты
